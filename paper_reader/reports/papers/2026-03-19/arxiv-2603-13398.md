# Qianfan-OCR: A Unified End-to-End Model for Document Intelligence

## Research Questions

- How can end-to-end vision-language models achieve OCR performance competitive with specialized multi-stage pipeline systems?
- How can explicit layout analysis capabilities be recovered in end-to-end OCR models that typically lose spatial structure information?
- What are the performance trade-offs between unified end-to-end models versus traditional OCR-then-LLM pipeline approaches for document understanding tasks?

## How This Paper Solves It

Traditional OCR systems use multi-stage pipelines (detection → recognition → understanding) that suffer from error propagation and lose visual context during text extraction. End-to-end models avoid these issues but sacrifice explicit layout analysis that users need for spatial grounding. Qianfan-OCR introduces "Layout-as-Thought" - an optional reasoning phase triggered by <think> tokens where the model first generates structured layout representations (bounding boxes, element types, reading order) before producing final outputs. This recovers layout capabilities while maintaining end-to-end optimization and improving accuracy on complex documents where structural priors help resolve ambiguities.

## Key Contributions

- First 4B-parameter end-to-end model unifying document parsing, layout analysis, and understanding in a single vision-language architecture
- Layout-as-Thought mechanism that recovers explicit layout analysis in end-to-end OCR through optional structured reasoning triggered by special tokens
- State-of-the-art results among end-to-end models on OmniDocBench v1.5 (93.12) and OlmOCR Bench (79.8), with highest average score on key information extraction benchmarks
- Comprehensive evaluation showing two-stage OCR+LLM pipelines degrade substantially on tasks requiring spatial reasoning (zero accuracy on chart interpretation)
- Large-scale data synthesis pipelines covering 6 domains with specialized augmentation strategies for OCR tasks

## Limitations of the Proposed Solution

- Layout-as-Thought only validated on document parsing tasks, effectiveness on KIE, chart understanding, and document QA remains unexplored
- 4B parameter footprint limits deployment in resource-constrained environments like edge devices and CPU-only servers
- Performance ceiling of purely end-to-end architectures versus heavily optimized pipeline systems remains unclear
- Current Layout-as-Thought generates rigid format layout dumps rather than flexible task-adaptive spatial reasoning

## Conclusions

- Qianfan-OCR ranks first among end-to-end models on OmniDocBench v1.5 (93.12) and OlmOCR Bench (79.8), achieving competitive performance with pipeline systems
- Layout-as-Thought improves accuracy on complex documents with high layout entropy while being counterproductive on simple documents
- Two-stage OCR+LLM pipelines show zero accuracy on chart interpretation benchmarks where spatial information is essential, demonstrating the value of preserving visual context
- With W8A8 quantization, Qianfan-OCR achieves 1.024 pages per second throughput, comparable to specialized pipeline systems despite having a 4B parameter backbone

## General Limitations

- Layout-as-Thought only validated on document parsing tasks, effectiveness on KIE, chart understanding, and document QA remains unexplored
- 4B parameter footprint limits deployment in resource-constrained environments like edge devices and CPU-only servers
- Performance ceiling of purely end-to-end architectures versus heavily optimized pipeline systems remains unclear
- Current Layout-as-Thought generates rigid format layout dumps rather than flexible task-adaptive spatial reasoning

## Abstract

We present Qianfan-OCR, a 4B-parameter end-to-end vision-language model that unifies document parsing, layout analysis, and document understanding within a single architecture. It performs direct image-to-Markdown conversion and supports diverse prompt-driven tasks including table extraction, chart understanding, document QA, and key information extraction. To address the loss of explicit layout analysis in end-to-end OCR, we propose Layout-as-Thought, an optional thinking phase triggered by special think tokens that generates structured layout representations -- bounding boxes, element types, and reading order -- before producing final outputs, recovering layout grounding capabilities while improving accuracy on complex layouts. Qianfan-OCR ranks first among end-to-end models on OmniDocBench v1.5 (93.12) and OlmOCR Bench (79.8), achieves competitive results on OCRBench, CCOCR, DocVQA, and ChartQA against general VLMs of comparable scale, and attains the highest average score on public key information extraction benchmarks, surpassing Gemini-3.1-Pro, Seed-2.0, and Qwen3-VL-235B. The model is publicly accessible via the Baidu AI Cloud Qianfan platform.
